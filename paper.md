**Abstract**

Formal theorem proving in environments like Lean 4 is fundamentally bottlenecked by sparse binary rewards and combinatorial state-space explosion. Existing approaches either rely on myopic tactic-level search or computationally expensive zero-shot generation. We present a novel hierarchical Reinforcement Learning framework that models proof generation as an asymmetric AND/OR search graph, explicitly decoupling high-level structural planning from low-level tactic execution. To mitigate the sparse reward problem and frequent syntax hallucinations, we introduce an AST-guided Auto-Sorrifier. This environment-integrated patching mechanism converts syntactically invalid generations into compiling partial proofs, yielding continuous, dense reward signals for RL training. Furthermore, we optimize the policy using Group Relative Policy Optimization (GRPO) augmented with a dependency-aware credit assignment mechanism. By tracing the Lean 4 kernel's formal dependency graph, our system provides precise, bottom-up structural rewards that penalize logical hallucinations and unused subgoals. Finally, by asymmetrically allocating the search budget to prioritize tactic resolution over structural branching, our approach maximizes parallel rollout throughput and strictly bounds tree expansion, significantly improving the sample efficiency of RL-driven theorem provers.

# 1. Introduction

Automated theorem proving in formal environments like Lean 4 requires generating exact logical steps and maintaining strict syntactic correctness. Current approaches typically follow two common paradigms: exploring proof states via tactic-level search (e.g., Monte Carlo Tree Search), or prompting Large Language Models (LLMs) to generate complete proofs in a single zero-shot pass.

Both methods face significant limitations. Tactic-level search creates a massive state space and struggles with long-horizon planning, making it difficult to discover necessary intermediate lemmas. Conversely, relying on LLMs for whole-proof generation requires high inference compute and lacks the efficiency of incremental state exploration. Furthermore, optimizing these models via Reinforcement Learning (RL) is heavily constrained by the environment's reward structure. Lean 4 provides a sparse, binary signal (e.g., 1 if the entire proof compiles, 0 otherwise). This binary outcome offers no credit for partially correct reasoning, limiting the effectiveness of RL optimization.

To address these issues, we propose an approach that explicitly separates proof search into two distinct operations: decomposing a goal into sub-hypotheses (generating proof skeletons) and resolving a goal directly (applying terminal tactics). We model this hierarchical process using an AND/OR search graph.

To overcome the sparse reward problem and handle the frequent syntax errors generated by LLMs, we introduce the AST-Guided Auto-Sorrifier. When an LLM generates a syntactically invalid step, this module parses the Abstract Syntax Tree (AST) to isolate the error and iteratively replaces the failing code with the `sorry` tactic. This mechanism allows the partial proof to compile within Lean. By doing so, it converts the binary pass/fail outcome into a dense reward signal, calculated based on the proportion of syntactically valid code remaining after patching.

We utilize this dense feedback to train a reasoning LLM using Group Relative Policy Optimization (GRPO). Because tactic execution and skeleton decomposition exhibit different evaluation characteristics, we optimize them in decoupled groups. Additionally, to discourage invalid decompositions, we apply a credit assignment mechanism that penalizes subgoals that are either logically invalid or unused in valid proof constructions.

In summary, our main contributions are:
* We introduce an AST-guided error patching mechanism that isolates syntax failures, converting sparse binary outcomes into dense reward signals for RL training.
* We propose an AND/OR search framework that explicitly separates high-level decomposition from tactic-level resolution.
* We develop a decoupled GRPO training pipeline utilizing a dependency-aware reward formulation that penalizes logical hallucinations.

# 2. Problem Formulation: The Asymmetric AND/OR Proof Graph

We formulate theorem proving in Lean 4 as a Markov Decision Process (MDP) modeled via a directed, bipartite AND/OR search graph. This graph captures the interaction between the policy (the LLM) and the formal verification environment (Lean 4 kernel). The system is defined by the tuple $\langle \mathcal{S}, \mathcal{A}, \mathcal{T}, \mathcal{R} \rangle$, where the reward formulation $\mathcal{R}$ is detailed in Section 4. The state space $\mathcal{S}$ and action space $\mathcal{A}$ correspond to two alternating node types:

* **OR-nodes (States $\mathcal{S}$):**
  An OR-node corresponds to a proof state $s = (\Gamma, g) \in \mathcal{S}$, where $g$ is the current goal and $\Gamma$ denotes the local context of available variables and hypotheses. Since each subgoal inherits only the context valid within its own branch, two nodes sharing the same goal $g$ but different contexts $\Gamma$ are treated as distinct states. An OR-node is considered *solved* if at least one of its outgoing actions successfully proves the goal. The root node is the initial theorem state $s_0 = (\Gamma_0, g_0)$, where $\Gamma_0$ contains the initial premises and variables, and $g_0$ is the main statement.

* **AND-nodes (Actions $\mathcal{A}$):**
  An AND-node corresponds to an action $a \sim \pi_\theta(\cdot \mid s)$, where each action consists of a sequence of tactics rather than a single step. To capture the hierarchical structure of mathematical reasoning, we partition the action space asymmetrically into two disjoint subsets, $\mathcal{A} = \mathcal{A}_{tac} \cup \mathcal{A}_{skel}$:
  * **Terminal Tactics ($a \in \mathcal{A}_{tac}$):**
    Sequences of execution tactics intended to fully discharge the goal (e.g., `intro x; simp; linarith`).
  * **Proof Skeletons ($a \in \mathcal{A}_{skel}$):**
    Sequences of structural steps (e.g., multiple `have` declarations followed by an `induction`) that decompose the goal into intermediate subgoals.

  An AND-node is *solved* only if all of its resulting child OR-nodes are solved. If a terminal sequence $a \in \mathcal{A}_{tac}$ successfully closes the goal, it produces no child nodes and is therefore solved immediately.

**Transition Dynamics ($\mathcal{T}$) and Boundary Conditions:**
The transition function $\mathcal{T}(s, a)$ is deterministically evaluated by the Lean 4 REPL. Given a state $s$ and a tactic sequence $a$, executing $a$ produces a set of resulting subgoals $\mathcal{T}(s, a) = \{s'_1, s'_2, \dots, s'_k\}$.

To control state space growth, we treat failures in proof skeletons and terminal tactics differently:

* **Skeleton Transitions:**
  If a proof skeleton ($a \in \mathcal{A}_{skel}$) contains errors, the Auto-Sorrifier replaces invalid segments with `sorry`, preserving the valid structural decomposition. The resulting subgoals ($k \ge 1$) are added as child OR-nodes.

* **Tactic Transitions:**
  Terminal tactics ($a \in \mathcal{A}_{tac}$) aim to fully discharge the goal. If a sequence fails, the Auto-Sorrifier may patch it with `sorry`, but each `sorry` introduces an unresolved subgoal. Treating such patched sequences as new nodes would lead to a large branching factor.

  To avoid this, failed tactic sequences do not generate child nodes. Instead, they are handled within the current node: we first sample half of the tactic budget, and if no sequence succeeds, we repair them using the Auto-Sorrifier and feed the patched sequences back to the policy model $\pi_\theta$ as in-context guidance for the remaining budget.

If a tactic sequence successfully closes the goal, no child nodes are produced ($k = 0$). If all attempts fail, the node is marked as a local failure and the branch is pruned.

Importantly, since only proof skeletons introduce new subgoals, the effective branching factor, and thus the search complexity, is governed by $\mathcal{A}_{skel}$, while terminal tactics only affect local resolution within a node.

# 3. Policy-Guided Level-wise Rollout

Standard search algorithms sequentially expand the search tree by selecting a single node at a time. While effective, sequential expansion underutilizes the parallel architecture of modern GPUs and autoregressive LLMs. To maximize data synthesis throughput for RL training while preventing objective mismatch between search exploration and policy optimization, we structure our trajectory generation as a **policy-guided level-wise rollout**.

Unlike uninformed search methods that force uniform exploration across all branches, our approach strictly samples actions from the active policy distribution $a \sim \pi_\theta(\cdot \mid s)$. At each search depth $d$, the system collects all active, unsolved OR-nodes (denoted as $B_d$). The policy model simultaneously samples a budget of $K$ candidate actions for each node. This constructs a global batch of size $B_d \times K$, allowing the LLM to process all nodes at the current depth in a single, highly parallel inference pass while naturally narrowing the search space toward high-value regions as the policy improves.

## 3.1. Asymmetric Action Allocation

To balance direct goal solving with goal decomposition, we apply an asymmetric allocation to the $K$ samples per node (e.g., 80% for terminal tactics and 20% for proof skeletons). This skewed distribution reflects the differing complexities of the search space:

* **Terminal Tactic Space:** The action space for tactic-level resolution is combinatorially large. A single goal can be solved using numerous tactic sequences or rewrite rules. Consequently, directly closing a goal requires broad sampling to find a valid compiling proof.

* **Proof Skeleton Space:** In contrast, the space of valid goal decompositions is strictly constrained, and a goal typically admits only a limited number of viable high-level proof strategies (e.g., induction or case analysis).

Allocating a smaller fraction of the budget to proof skeletons is empirically sufficient to capture the key decomposition steps. By prioritizing terminal tactics, the system increases the likelihood of closing goals while avoiding excessive branching from unnecessary subgoal generation.

## 3.2. Complexity and Resource Management

We formalize the algorithmic complexity of the level-wise rollout. To keep notation simple, let $D$ be the maximum allowed search depth, $N$ be the absolute node expansion limit (safety timeout), $M$ be the maximum hardware micro-batch size, $L$ be the maximum token generation length per action, and $c$ be the average Lean 4 verification time.

* **Search Bounding and Micro-batching:** Unlike standard OR-trees, pruning active OR-nodes at a given depth breaks the strict logical dependency of parent AND-nodes, causing cascading structural failures. Therefore, all $B_d$ active nodes at depth $d$ must be evaluated. To prevent Out-Of-Memory (OOM) errors when the required inference batch $B_d \times K$ exceeds VRAM limits, the generation is partitioned into sequential micro-batches of size $M$.

* **Global Truncation:** To prevent intractable combinatorial explosion, the search is bounded by two global limits: a maximum depth $D$ and an absolute node expansion limit $N$. If a proof attempt exceeds depth $D$ or the total tree size surpasses $N$, the rollout episode is immediately truncated. All unresolved OR-nodes at the truncation boundary are marked as incomplete. The specific credit assignment and penalty mechanisms for these truncated trajectories are handled by the reward formulation (see Section 6).

* **Time Complexity:** At any depth $d$, generating $K$ actions for $B_d$ nodes requires $O(B_d \cdot K \cdot L)$ inference time, and Lean verification takes $O(B_d \cdot K \cdot c)$. Because the total number of expanded nodes across all depths is strictly bounded by the safety limit $N$ (i.e., $\sum B_d \le N$), the maximum execution time per theorem is strictly bounded by $O(N \cdot K \cdot (L + c))$.

* **Space Complexity:** The structural memory required to maintain the logical dependency graph is $O(N \cdot K)$. Crucially, due to the micro-batching mechanism, the LLM KV-cache memory footprint is capped at $O(M \cdot L)$. This completely decouples the peak VRAM utilization from the graph's breadth $B_d$, guaranteeing safe execution on hardware.

# 4. Policy Action Generation and LLM Interface

To connect the MDP formulation described in Section 2 with the LLM policy $\pi_\theta$, we map the formal state $s = (\Gamma, g)$ into a textual representation. The policy uses a single model to generate both terminal tactics ($\mathcal{A}_{tac}$) and proof skeletons ($\mathcal{A}_{skel}$), distinguished entirely via prompts.

## 4.1. State Serialization

We serialize the Lean 4 proof state into a string format that mirrors Lean's native Infoview. The local context $\Gamma$ (variable declarations and hypotheses) is listed first, followed by the target goal $g$ (e.g., `$\vdash$ Q`). This `[GOAL STATE]` block serves as the prefix for the prompt. During batched rollouts, keeping this state prefix identical across multiple sampled actions allows the inference engine to reuse the KV-cache, improving generation throughput.

## 4.2. Task-Specific Prompting and Chain-of-Thought

Rather than training separate models for different action types, we use a single policy $\pi_\theta$ conditioned on task-specific prompts. To encourage intermediate planning, we apply a standard Chain-of-Thought (CoT) format: the prompt instructs the model to generate its reasoning within a `<think>` block before outputting the formal Lean code inside a `lean` block.

Depending on which action type is being sampled, the prompt includes one of two directives:

* **Terminal Tactic Generation:** The prompt includes `[TASK: TERMINAL TACTIC]`. The model is instructed to generate a tactic sequence that fully closes the goal without using the `sorry` placeholders (e.g., using solvers like `simp`, `omega`, or `linarith`).

* **Proof Skeleton Generation:** The prompt includes `[TASK: PROOF SKELETON]`. The model is instructed to decompose the goal into intermediate steps (e.g., using `have`, `induction`, or `case`) leaving the subgoals unproven via the `sorry` tactic.       

By separating the action space through these distinct prompts, the RL algorithm can independently optimize the generation behavior for $a \in \mathcal{A}_{tac}$ and $a \in \mathcal{A}_{skel}$ within the same policy.

# 5. AST-Guided Auto-Sorrifier

Language models frequently generate syntactically invalid code when producing formal proofs. Standard search algorithms typically discard such invalid generations, resulting in significant sample inefficiency. To address this issue, we introduce the AST-Guided Auto-Sorrifier, an iterative error-patching mechanism tightly integrated with the Lean 4 execution environment.

Instead of discarding a failed generation, the Auto-Sorrifier retains the syntactically valid portions of the generated code by replacing erroneous segments with the `sorry` tactic—a built-in placeholder in Lean 4 that allows incomplete proofs to type-check.

## 5.1. AST Extraction and Environment Caching
A primary engineering bottleneck in Lean 4 is the substantial computational overhead required to load foundational libraries (e.g., `Mathlib`) during each verification step. To decouple AST extraction from the REPL verification latency, we implement a persistent background Lean 4 process. This process caches the compiled `Environment` based on the target file's import header. By reusing this environment, the system bypasses redundant initialization, rapidly parsing the source code into an Abstract Syntax Tree (AST) and mapping exact byte offsets to syntax node boundaries in milliseconds.

## 5.2. Error Isolation and Iterative Patching
When the Lean REPL reports a compilation error or an unsolved goal, the system cross-references the error's reported line position with the extracted AST blocks. The Auto-Sorrifier employs an innermost-first strategy: it identifies the narrowest enclosing syntax node (e.g., a specific leaf tactic or a term-level expression) and replaces it with `sorry`. This minimal-intervention approach ensures that surrounding valid logic remains strictly preserved. For faulty declarations, it intelligently truncates the body (e.g., rewriting `:= \dots` to `:= by sorry`). The modified code is then re-submitted to the REPL, and this patching process iterates until the file compiles successfully.

## 5.3. Oscillation Detection and Heuristic Fallback
Due to the strictness of Lean's parser, local syntax mutations can occasionally induce cascading parsing failures, leading the iterative patching loop into an infinite state oscillation. To guarantee termination within a fixed computational budget, the Auto-Sorrifier tracks the cryptographic hash of the file content across iterations. If an exact state oscillation is detected, the system abandons the AST-based local replacement and triggers a structural fallback. It utilizes indentation-based heuristics to scan backward, locating the nearest structural parent block (e.g., `have`, `lemma`, or `theorem` declarations). The system then replaces the entire parent block with `sorry` and aggressively prunes its deeply indented child commands.

By systematically isolating and replacing errors, the Auto-Sorrifier prevents local syntax failures from causing an entire proof branch to be discarded. This robust mechanism ensures that valid tactic prefixes and structural decompositions are preserved, enabling the extraction of continuous, dense reward signals based on partial correctness.

# 6. Dense Reward Formulation and Credit Assignment

Standard RL methods typically propagate a uniform scalar reward across an entire trajectory. This approach fails to assign accurate credit in hierarchical theorem proving, where a single action (e.g., a goal decomposition) can produce multiple subgoals with entirely different roles—some necessary, some redundant, and some unprovable. To address this, we compute the return by explicitly analyzing how each generated subgoal contributes to the final proof structure. We define two levels of reward: an immediate syntactic reward that measures whether the generated code compiles, and a delayed structural reward evaluated via a hindsight tree backup on the AND/OR graph.

## 6.1. Immediate Syntactic Reward

When the policy model generates an action $a$ (either a proof skeleton or a terminal tactic sequence) for a state $s$, the Auto-Sorrifier patches syntax errors to ensure compilation. However, to prevent the model from inflating its reward via trivial text generation, we formulate the reward calculation strictly over semantic elements.

First, we parse the generated action $a$ into an Abstract Syntax Tree (AST) and explicitly filter out all non-semantic tokens (e.g., comments, whitespace, and redundant delimiters). Let $\mathcal{T}_{total}$ denote the set of remaining semantic AST nodes (e.g., tactic invocations, term declarations) that the model actively attempted to execute.

Next, we apply the Auto-Sorrifier to patch errors and utilize Lean's built-in linter to prune dead code—any valid node that does not logically modify the local context or goal state. Let $\mathcal{T}_{valid} \subseteq \mathcal{T}_{total}$ denote the subset of these semantic nodes that compile successfully without being replaced by `sorry`, and actively contribute to the proof state.

We explicitly formulate the immediate syntactic reward $r_{env}(s, a) \in [0, 1]$ as:

$$
r_{env}(s, a) =
\begin{cases}
\frac{|\mathcal{T}_{valid}|}{|\mathcal{T}_{total}|} & \text{if } |\mathcal{T}_{total}| > 0 \\
0 & \text{otherwise}
\end{cases}
$$

An action that compiles perfectly and contains no dead code yields $\mathcal{T}_{valid} = \mathcal{T}_{total}$, resulting in $r_{env} = 1.0$. Conversely, an action that reduces entirely to syntax errors or dead code yields $|\mathcal{T}_{valid}| = 0$, resulting in $r_{env} = 0.0$.

## 6.2. Dependency-Aware Structural Reward

For proof skeletons, local syntactic validity is insufficient; the generated sub-hypotheses must be logically necessary. Once a search episode concludes (either by successfully proving the root theorem or hitting the expansion limit), we evaluate the trajectory using Lean's kernel dependency graph.

The dependency graph traces the logical lineage from the final closed goal backward to the hypotheses that were actively invoked. We classify the subgoals generated by a skeleton into a partitioned set $H = \bigcup_{i \in \{c, b, m\}} H_i$, representing their distinct logical roles:

* **Core Dependency ($H_c$):** Hypotheses explicitly required and used to close the target goal.
* **Benign Redundancy ($H_b$):** Hypotheses that are formally provable but logically ignored in the final proof.
* **Malignant Hallucination ($H_m$):** Subgoals that cannot be proven within the formal environment.

We assign specific reward weights $W_i$ to each corresponding hypothesis partition, strictly satisfying the penalty hierarchy $W_m < W_b \le 0 < W_c = 1$. The intrinsic logical reward for the decomposition action itself, denoted as $r_{dep}(s, a)$, is defined as the weighted ratio of these partitions:

$$r_{dep}(s, a) = \sum_{i \in \{c, b, m\}} W_i \frac{|H_i|}{|H|}$$

This formulation ensures that generating a skeleton with redundant or unprovable subgoals explicitly penalizes the action's intrinsic value, quantifying the efficiency of the decomposition step.

## 6.3. Deterministic AND/OR Tree Backup

To propagate this credit through the hierarchical AND/OR graph without introducing additional variance into the GRPO objective, we perform a deterministic AND/OR tree backup. Let $\mathbb{1}_{\text{solved}}(s,a) \in \{0,1\}$ be a binary indicator denoting whether the subgraph rooted at action $a$ is successfully closed. The resulting action-value $Q(s,a)$ and state-value $V(s)$ are then computed recursively from the leaf nodes up to the root:

**1. Terminal Tactics ($a \in \mathcal{A}_{tac}$):**

Tactics do not generate subgoals. Their value is the immediate syntactic reward plus a strict scalar bonus $W_{solve}$ if they successfully discharge the goal:

$$Q(s,a) = r_{env}(s,a) + W_{solve} \cdot \mathbb{1}_{\text{solved}}(s, a)$$

**2. State Value (OR-nodes):**

The value of a state $s$ is entirely determined by its most successful outgoing action. This $\max$ operator naturally filters out the noise of failed search branches, protecting the GRPO signal from high-variance exploration trajectories:

$$V(s) = \max_{a \in \mathcal{A}(s)} Q(s,a)$$

**3. Proof Skeletons ($a \in \mathcal{A}_{skel}$):**

A skeleton generates a set of subgoals $S'$. As an AND-node, it successfully closes only if *all* subgoals in $S'$ are solved. We enforce this strict logical conjunction using the $\min$ operator. The action value follows a deterministic Bellman backup, summing the syntactic reward, the intrinsic dependency reward, and the discounted future value of its worst-case subgoal:

$$Q(s,a) = r_{env}(s,a) + \mathbb{1}_{\text{solved}}(s, a) \cdot \Big( r_{dep}(s, a) + \gamma \min_{s' \in S'} V(s') \Big)$$

This final $Q(s,a)$ serves directly as the deterministic return $R_i$ for each sampled action in the GRPO optimization group, ensuring stable, low-variance policy updates grounded in global logical correctness.

## 6.4. Mathematical Robustness Against Reward Hacking

A critical vulnerability in optimizing formal theorem provers via continuous rewards is the "error dilution" exploit. A policy model acting as a rational agent might attempt to mask syntax errors (which heavily penalize $r_{env}$) by generating an arbitrarily large number of valid but redundant operations. We mathematically demonstrate why the structural penalty weights must be strictly bounded to prevent this asymptotic hacking.

Let an action $a \in \mathcal{A}_{skel}$ generate a total of $N_{total}$ parsed AST nodes, distributed across syntactic errors $N_e \ge 1$, core dependencies $N_c = |H_c| \ge 1$, and benign redundancies $N_b = |H_b| \ge 0$. Assuming a fully rational agent seeking to maximize its return, it will avoid generating malignant hallucinations, hence $N_m = |H_m| = 0$.

Given a successfully closed parent goal ($\mathbb{1}_{\text{solved}} = 1$), the agent receives the discounted future value of its subgoals. Let $V_{future} = \gamma \min_{s' \in S'} V(s')$. Because a rational hacking agent exploits the system by generating trivially true benign subgoals (e.g., `1 = 1` or `True`), the value of these redundant nodes approaches the maximum reward. Consequently, the minimum value of the entire AND-node formulation is strictly bottlenecked by the necessary core subgoals ($\min V(S_{core}) \ll \min V(S_{benign})$). Thus, the $\min$ operator natively filters out the benign padding, rendering $V_{future}$ a constant independent of $N_b$.

Assuming the core hypothesis weight $W_c = 1$, the total action-value $Q(N_b)$ as a function of the spammed benign nodes $N_b$ is:

$$Q(N_b) = \underbrace{\frac{N_c + N_b}{N_c + N_b + N_e}}_{r_{env}(N_b)} + \underbrace{\frac{N_c + W_b N_b}{N_c + N_b}}_{r_{dep}(N_b)} + V_{future}$$

**The Universal Robustness Constraint:**

To strictly disincentivize hacking, the honest return $Q(0)$ (where no redundant nodes are generated) must strictly exceed the hacking return $Q(N_b)$ for any arbitrary number of spammed nodes $N_b > 0$:

$$Q(0) > Q(N_b)$$

$$\frac{N_c}{N_c + N_e} + 1 > \frac{N_c + N_b}{N_c + N_b + N_e} + \frac{N_c + W_b N_b}{N_c + N_b}$$

Through algebraic isolation of $W_b$ (subtracting terms and dividing by $N_b > 0$), we establish the strict upper bound function for $W_b$, denoted as $f(N_b, N_e)$:

$$W_b < 1 - \frac{N_e (N_c + N_b)}{(N_c + N_e)(N_c + N_b + N_e)} = f(N_b, N_e)$$

> **Theorem (Non-Positive Weight Constraint)**
>
> To guarantee global robustness, the structural weight $W_b$ must strictly satisfy $W_b < f(N_b, N_e)$ for all possible adversarial conditions ($N_b > 0$ and $N_e \ge 1$).
>
> By the properties of limits in real analysis, if a constant is strictly less than a function across its domain, it must be less than or equal to the function's infimum. Thus, we evaluate the limits to find the absolute bound:
>
> $$W_b \le \inf_{N_b > 0, N_e \ge 1} f(N_b, N_e)$$
>
> First, because the function is strictly decreasing with respect to the spam length $N_b$, we take the limit as $N_b \to \infty$:
>
> $$W_b \le \lim_{N_b \to \infty} f(N_b, N_e)$$
>
> $$W_b \le 1 - \frac{N_e}{N_c + N_e} = \frac{N_c}{N_c + N_e}$$
>
> Second, this upper bound must universally hold for any magnitude of syntax errors $N_e \ge 1$. We take the limit as $N_e \to \infty$:
>
> $$W_b \le \lim_{N_e \to \infty} \frac{N_c}{N_c + N_e}$$
>
> $$W_b \le 0$$
>
> **Conclusion:** To mathematically guarantee that a policy cannot exploit the syntactic ratio $r_{env}$ by generating infinite redundant subgoals, we must strictly enforce $W_b \le 0$ (and consequently $W_m < 0$). By establishing this non-positive constraint, the artificial inflation of the syntactic ratio is mathematically dominated by the decay of the structural reward, ensuring that generating unnecessary subgoals yields a strictly suboptimal return.

When a search episode is truncated due to depth or expansion limits, the policy might exploit this boundary by intentionally stalling to receive a neutral score, thereby avoiding negative structural penalties ($W_b$ or $W_m$). To mathematically neutralize this reward hacking, all unresolved OR-nodes at the truncation boundary are strictly penalized as malignant failures. However, the system retains all dense syntactic rewards ($r_{env}$) and any local structural rewards from subgraphs completely resolved prior to the timeout. This strict penalty mechanism forces the optimization landscape to prioritize concise, complete proofs over stalling tactics.

To mathematically guarantee that a policy cannot exploit the syntactic ratio $r_{env}$ by generating infinite redundant subgoals, we must strictly enforce $W_b \le 0$ (and consequently $W_m \ll 0$). By establishing this non-positive constraint, the artificial inflation of the syntactic ratio is completely neutralized by the monotonous decay of the structural reward, ensuring that generating unnecessary subgoals yields a strictly suboptimal return.

# 7. RL Optimization via GRPO

We optimize the policy model using Group Relative Policy Optimization (GRPO) on trajectories generated during batched search. Because terminal tactics and proof skeletons are evaluated with different reward signals, comparing them within the same group leads to inconsistent training signals. To address this, we train them separately by grouping actions based on their type.

## 7.1. Tactic Group Optimization

For terminal tactics, actions are grouped per state. The advantage for each sample is computed relative to other samples from the same state using the immediate reward $r_{env}$.

To maintain stability, we exclude groups where all samples receive identical rewards ($std(R) = 0$). In such cases, no informative learning signal is available, and including them may introduce numerical instability.

## 7.2. Skeleton Group Optimization

For proof skeletons, actions are also grouped per state. Their quality is evaluated using the dependency-aware value $Q(s, a)$, which reflects whether the generated subgoals are necessary and provable. This encourages the model to produce concise decompositions while avoiding redundant or unprovable subgoals.

Training with these signals improves the policy’s ability to generate relevant actions. During inference, this results in a more focused search that explores fewer irrelevant branches, reducing computational cost while preserving the correctness guarantees of the AND/OR framework.

## 7.3. GRPO Objective and KL-Divergence Regularization

We optimize the policy model $\pi_{\theta}$ using the Group Relative Policy Optimization (GRPO) objective. For each state $s$, we sample a group of $K$ actions (either tactics or skeletons). Let $A_i$ be the normalized advantage of action $a_i$ within its specific group, computed using our decoupled reward formulations.To prevent destructive policy updates, we bound the policy ratio using a clipping function and apply a Kullback-Leibler (KL) divergence penalty against the frozen reference model $\pi_{ref}$. The loss function is explicitly defined as:

$$J_{GRPO}(\theta) = \mathbb{E}_{s \sim P, a_i \sim \pi_{\theta}} \left[ \frac{1}{K} \sum_{i=1}^{K} \left( \min \left( \frac{\pi_{\theta}(a_i|s)}{\pi_{ref}(a_i|s)} A_i, \text{clip}\left(\frac{\pi_{\theta}(a_i|s)}{\pi_{ref}(a_i|s)}, 1-\epsilon, 1+\epsilon\right) A_i \right) - \beta D_{KL}(\pi_{\theta} || \pi_{ref}) \right) \right]$$

where $\epsilon$ is the clipping hyperparameter (e.g., $0.2$) and $\beta$ is the KL penalty coefficient. To maintain computational efficiency during RL training, the KL divergence is approximated unbiasedly using the log-ratio of the action probabilities:

$$D_{KL}(\pi_{\theta} || \pi_{ref}) \approx \log \frac{\pi_{ref}(a_i|s)}{\pi_{\theta}(a_i|s)}$$

This explicit regularization ensures that the optimizing policy does not collapse or deviate excessively from its pre-trained reasoning capabilities during the pursuit of dense structural rewards.

# Appendix A: System Assumptions and Empirical Justifications

Our hierarchical AND/OR search and dense reward formulation rely on several underlying assumptions about the formal environment and the policy model. In this section, we explicitly state these assumptions and provide theoretical and empirical justifications supporting our system design.

## A. Assumption List

**A1.** **Syntactic validity provides a useful local signal:** Syntactically valid code (after AST-guided patching) carries informative signal for local progress within the proof state.
**A2.** **Executable prefixes are informative:** The valid prefix of a partially correct action reflects meaningful structure that can be leveraged for partial credit assignment.
**A3.** **Auto-Sorrifier preserves useful structure:** Replacing erroneous fragments with `sorry` retains sufficient structural information for downstream evaluation and learning.
**A4.** **Dependency reflects contribution:** The dependency graph of a proof term reflects the contribution of intermediate hypotheses within that specific proof instance.
**A5.** **Dependency is sufficiently discriminative:** The dependency structure enables meaningful differentiation between core, unused, and unprovable subgoals.
**A6.** **Reward weighting induces desired behavior:** The combination of syntactic and structural rewards, together with weight constraints, promotes desirable proof construction behavior.
**A7.** **Structural reward is reachable:** During search, successful proof trajectories occur with non-zero probability, enabling propagation of delayed reward signals.
**A8.** **Decomposition simplifies subgoals:** Goal decomposition produces subgoals that are, on average, more tractable than the original goal.
**A9.** **Model can generate useful decompositions:** The policy model is capable of generating decomposition steps that lead to meaningful and solvable subgoals.
**A10.** **Search budget is sufficient:** The allocated search budget allows exploration of a sufficiently diverse set of candidate actions.
**A11.** **Group-wise reward differences are informative:** Within each training group, reward differences are sufficient to provide meaningful learning signals for GRPO.
**A12.** **Reward aligns with proof success:** The reward function correlates with the likelihood of successfully completing a proof.

**A14.** **Action space factorization is appropriate:** Separating decomposition and tactic execution is a suitable abstraction for modeling the proof search process.
**A15.** **Asymmetric allocation is reasonable:** An asymmetric allocation of search effort between decomposition and tactic execution provides an effective exploration balance.
**A16.** **Partial correctness is learnable:** The model can learn from partially correct intermediate states through continuous reward signals.

## B. Assumption Groups and Justifications

### B.1. Syntactic Validity as a Local Signal (Assumptions 1, 2, 3, 16)

**The Premise:** We assume that local syntactic validity (after AST-guided patching) correlates meaningfully with proof progress.

**Justification:** In Lean 4, relying on the propositions-as-types principle, syntactic validity is a necessary local signal, though not a sufficient correctness certificate. While the Auto-Sorrifier's patching does not guarantee global logical soundness, passing the strict type-checker confirms that the generated expressions adhere to the local context ($\Gamma$). Therefore, the valid prefix and the patched AST nodes serve as informative surrogate signals, allowing the model to learn partial correctness without being entirely blind to the global proof structure.

### B.2. Dependency Graph as an Attribution Proxy (Assumptions 4, 5, 13)

**The Premise:** We assume the proof term's dependency graph reflects the contribution of individual subgoals, allowing us to discriminate between core, benign, and malignant hypotheses.

**Justification:** The dependency graph extracted from Lean’s kernel does not necessarily guarantee minimal necessity (as shorter, alternative proofs may exist). However, it provides a kernel-verified interpretation of the specific proof constructed by the model. If a hypothesis is consumed in the final proof term, it contributed to that particular resolution. Thus, the dependency graph serves as a deterministic attribution proxy for credit assignment, offering a more structured signal than purely statistical heuristics.

### B.3. Objective Design Against Degenerate Behaviors (Assumptions 6, 12, 18)

**The Premise:** We assume our penalty weights ($W_c, W_b, W_m$) encourage the desired behavior and discourage degenerate policies (reward hacking).

**Justification:** While it is not possible to guarantee immunity against all adversarial behaviors in deep RL, our objective is explicitly constructed to penalize a common family of degenerate behaviors: length-inflation (error dilution). By constraining $W_{benign} \le 0$, we establish an asymptotic limit that discourages the generation of redundant but valid subgoals. This biases the optimization landscape toward more concise goal resolution rather than artificial reward accumulation.

### B.4. Asymmetric Action Space and Heuristic Allocation (Assumptions 8, 9, 14, 15)

**The Premise:** We assume that separating decomposition from tactic execution is appropriate, and that an asymmetric search budget (e.g., 80%/20%) provides a reasonable exploration balance.

**Justification:** The action space for terminal tactics is combinatorially broader than the space of valid structural decompositions. Consequently, the 80%/20% split is deployed as an empirical heuristic to balance compute resources, rather than a rigid theoretical rule. Furthermore, while goal decomposition aims to reduce problem complexity, its practical effectiveness depends on the model's ability to generate useful subgoals. The hierarchical framework provides a structured way to separate and optimize these two reasoning modes.

### B.5. Empirical Prerequisites for RL Optimization (Assumptions 7, 10, 11, 17)

**The Premise:** We assume the delayed structural reward is reachable ($V_{core} = 1$) and that batched rollouts generate sufficient variance for informative GRPO updates.

**Justification:** The reachability of the structural reward (A7) is an empirical prerequisite of our system. If a policy fails to solve any goals, the sparse structural signal cannot bootstrap the learning process. To address this, our system is initialized from a warm-started reasoning model (e.g., DeepSeekMath or Llama-3-Instruct) with a non-zero baseline solve rate. Given such initialization, batched rollouts can generate sufficient intra-node variance to compute meaningful relative advantages, enabling stable policy improvement.
